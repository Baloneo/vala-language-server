LockStatement

Object Hierarchy:

Description:

public class LockStatement : CodeNode, Statement

Represents a lock statement e.g. 

 lock (a);
or 
 lock (a) { f(a); }
.

If the statement is empty, the mutex remains locked until a corresponding UnlockStatement occurs. Otherwise it's translated into a try/finally statement which unlocks the mutex after the block is finished.
